# 架构与执行流

**创建时间**: 20260504 09:30

> 实际已落地的架构，不是设计稿。后续改动请同步更新。

## 一、技术栈

| 层 | 选型 | 备注 |
|---|---|---|
| LLM | Anthropic Claude (`claude-sonnet-4-6`) | 走 `langchain-anthropic` |
| Embedding | `BAAI/bge-small-zh-v1.5` (sentence-transformers) | 本地，~100MB，需 HF 网络下载（首次） |
| 向量库 | ChromaDB (`langchain-chroma`) | 持久化目录 `chroma_db/` |
| Agent 框架 | LangChain 1.x `create_agent` + LangGraph 1.x | `create_agent` 内部已是 LangGraph |
| Web | FastAPI + uvicorn | 端口 8004 |
| 流式 | StreamingResponse + `agent.astream_events(version="v2")` | `text/event-stream` |
| 前端 | 单文件 HTML + 原生 JS（fetch + ReadableStream） | `app/web/index.html` |

## 二、四种 Agent 形态

| Agent | 实现 | 工具 | 记忆 | 何时挑选 |
|-------|------|------|------|---------|
| `single` | `create_agent` | calculator / current_time / mock_web_search | ✅ checkpointer thread | 通用对话、简单算数/查询 |
| `rag` | `create_agent` | basic + `retrieve_documents` | ✅ checkpointer thread | 问行业/研报/市场综述 |
| `plan` | LangGraph StateGraph | basic + retrieve（每步用子 agent） | ❌（当前未挂） | 复杂多步问题，需要先拆解 |
| `multi` | LangGraph StateGraph | researcher/analyst/writer 专精 | ❌（当前未挂） | 协作场景，需要多角色产出 |

## 三、关键执行流

### 3.1 single / rag agent

```
user msg
  → create_agent (LangGraph 内部图)
      └─ model 节点 ↔ tools 节点 循环
          直到模型不再 tool_call
  → AIMessage 返回
```

记忆通过 `config={"configurable": {"thread_id": ...}}` + InMemorySaver checkpointer 实现，
同 thread_id 自动恢复全部历史 messages。

### 3.2 plan-and-execute (`app/agents/planner.py`)

```
START
  → planner 节点：LLM 输出 JSON 步骤数组
  → executor 节点（循环 N 次，每次执行 plan[i]，结果存 step_results[i]）
       ↑——若还有未执行步骤
  → responder 节点：把 step_results 汇总成最终回答（流式）
  → END
```

### 3.3 supervisor 多 agent (`app/agents/multi_agent.py`)

```
START
  → supervisor 节点：用 with_structured_output 选一个角色或 FINISH
  → researcher / analyst / writer 任一节点执行
      ↑——执行完回到 supervisor
  → END（supervisor 选 FINISH 或达到 MAX_ITERATIONS=8）
```

防死循环：每个 supervisor tick `iterations += 1`，超过 8 强制结束。

## 四、SSE 事件协议

`POST /api/chat` 返回 `text/event-stream`，每行 `data: <json>\n\n`：

| `type` | 字段 | 说明 |
|--------|------|------|
| `token` | `content` | LLM 流式 token 片段 |
| `tool_start` | `name`, `input` | 工具开始调用 |
| `tool_end` | `name`, `output` (截断 1000) | 工具执行完毕 |
| `node` | `name`, `phase=start/end` | LangGraph 节点切换 |
| `done` | — | 整体结束 |
| `error` | `message` | 异常 |

前端 `app/web/index.html` 用 fetch + ReadableStream 解析，按类型渲染：
- `token` 追加到当前 assistant 气泡
- `tool_*` / `node` 写到旁边的事件框（虚线方框）
- `error` 红字高亮

## 六、已知局限（学习项目可接受）

1. plan / multi agent 暂无会话记忆 —— 改造方法：在 build_*() 里 `compile(checkpointer=...)` 即可，但 state 字段需匹配
2. RAG 用 cosine 相似度，未做重排序（rerank）
3. Web UI 没做对话历史持久化，刷新即丢
4. 没有鉴权、没有 rate limit
5. embedding 首次加载需 HF 网络（要打开代理）；下载到本机 HF 缓存后离线可用
